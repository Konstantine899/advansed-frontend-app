"# Руководство по деплою API для демонстрации

## Для демонстрации работодателю:

### Вариант 1: Использовать готовый API (рекомендуется)
1. API уже настроен и работает по адресу: `https://advansed-frontend-api.vercel.app`
2. Фронтенд автоматически использует этот URL при сборке для production

### Вариант 2: Развернуть свой API на Vercel
1. Создайте новый репозиторий на GitHub
2. Скопируйте папку `api/` в новый репозиторий
3. Зайдите на [Vercel.com](https://vercel.com)
4. Импортируйте репозиторий
5. Получите URL вашего API
6. Обновите URL в `package.json`:
   ```json
   \"build:prod\": \"webpack --env mode=production apiUrl=ВАШ_НОВЫЙ_URL\",
   ```

## Что будет работать в демо:
- ✅ **CRUD операции** (создание, чтение, обновление, удаление)
- ✅ **Авторизация** (логин: admin/123, user/123)
- ✅ **Изменения сохраняются** во время сессии
- ✅ **Все фичи приложения** работают

## Для демонстрации:
1. Запустите CI/CD пайплайн (автоматически при push в master)
2. Приложение задеплоится на GitHub Pages
3. API работает на Vercel
4. Работодатель может тестировать все функции

## Тестовые пользователи:
- **Админ:** username: `admin`, password: `123`
- **Пользователь:** username: `user`, password: `123`

## Важно:
- Изменения сохраняются только во время текущей сессии
- При перезапуске API данные сбрасываются (но это нормально для демо)
- Для production нужно использовать реальную базу данных"
