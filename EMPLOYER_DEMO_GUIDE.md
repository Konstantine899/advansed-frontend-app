# 🚀 Демонстрационное руководство для работодателя

## Обзор проекта

**Advansed Frontend App** - полнофункциональное React приложение с полным стеком технологий:
- **Frontend:** React + TypeScript + Redux Toolkit
- **Backend:** JSON Server API на Vercel
- **CI/CD:** GitHub Actions с автоматическим деплоем
- **Хостинг:** GitHub Pages (frontend) + Vercel (API)

## 🧪 Пошаговая демонстрация

### Шаг 1: Проверка CI/CD пайплайна
1. Перейдите в [GitHub Actions](https://github.com/Konstantine899/advansed-frontend-app/actions)
2. Откройте последний запуск workflow "CI/CD Pipeline"
3. Убедитесь, что все шаги выполнены успешно (зеленые галочки)

**Что проверяется в CI/CD:**
- ✅ Линтинг TypeScript и SCSS
- ✅ Юнит-тесты (84 теста)
- ✅ Сборка Webpack и Vite
- ✅ Деплой на GitHub Pages

### Шаг 2: Тестирование Frontend приложения
1. Откройте [Frontend приложение](https://konstantine899.github.io/advansed-frontend-app/)
2. Убедитесь, что приложение загрузилось без ошибок

### Шаг 3: Демонстрация CRUD операций

#### Авторизация:
1. Нажмите кнопку "Войти" в правом верхнем углу
2. Используйте тестовые данные:
   - **Администратор:** Логин `admin`, Пароль `123`
   - **Пользователь:** Логин `user`, Пароль `123`

#### Работа со статьями:
1. **Чтение (GET):** Просмотрите список статей на главной странице
2. **Создание (POST):** 
   - Нажмите "Создать статью"
   - Заполните форму и сохраните
   - Убедитесь, что статья появилась в списке
3. **Редактирование (PUT):**
   - Откройте любую статью
   - Нажмите "Редактировать"
   - Внесите изменения и сохраните
4. **Удаление (DELETE):**
   - Откройте статью
   - Нажмите "Удалить"
   - Подтвердите удаление

### Шаг 4: Проверка API
1. Откройте [API](https://advansed-frontend-app-xry5.vercel.app/)
2. Должна отобразиться страница JSON Server с доступными ресурсами
3. Проверьте health endpoint: https://advansed-frontend-app-xry5.vercel.app/health

### Шаг 5: Дополнительные функции

#### Интернационализация:
- Переключите язык в настройках профиля (русский/английский)
- Убедитесь, что интерфейс переводится

#### Темная тема:
- Переключите тему в настройках профиля

#### Адаптивный дизайн:
- Измените размер окна браузера
- Убедитесь, что интерфейс адаптируется

## 🛠 Технические детали

### Архитектура:
- **Feature-Sliced Design (FSD)** - модульная архитектура
- **Redux Toolkit** - управление состоянием
- **React Router** - навигация
- **i18next** - интернационализация

### Тестирование:
- **Jest** - юнит-тесты
- **Loki** - визуальное тестирование
- **ESLint/Prettier** - линтинг и форматирование

### Сборка и деплой:
- **Webpack** - production сборка
- **Vite** - development сервер
- **GitHub Actions** - CI/CD автоматизация

## 📊 Что демонстрирует проект

### Для Frontend разработчика:
1. **Современный стек:** React 18, TypeScript, Redux Toolkit
2. **Архитектура:** Чистая архитектура по FSD
3. **Тестирование:** Полное покрытие тестами

1. **API интеграция:** Работа с REST API

3. **Состояние:** Глобальное управление состоянием
4. **Оптимизация:** Code splitting, lazy loading

### Для DevOps/CI-CD:

2. **Деплой:** Мульти-платформенный деплой

4. **Мониторинг:** GitHub Actions с отчетностью

## 🐛 Известные ограничения

### В демо-режиме:
- Данные сохраняются только в текущей сессии
- При перезагрузке API данные сбрасываются
- Это нормально для демонстрации

### Для production нужно:
- Реальная база данных (MongoDB/PostgreSQL)
- Постоянное хранилище для данных
- Настоящая аутентификация с JWT
